# ABI Baselines

Committed JSON snapshots of each Soroban contract's exported function names and
contract-spec hash. CI fails when a built WASM differs from its baseline in a
breaking way (removed exports or changed spec hash) without an explicit
acknowledgement.

## Files

- `<contract-name>.json` — committed ABI baseline generated by `check_wasm_exports.py`.
- `<contract-name>.breaking.txt` — **optional** acknowledgement file. Create this to
  tell CI that the breaking change is deliberate and paired with a migration or
  major-version bump. The script updates the baseline on the next run.

## Updating baselines (non-breaking additions)

Baselines are updated automatically when new exports are detected (exit code 2).
Commit the updated JSON files.

## Acknowledging a deliberate breaking change

1. Verify the migration covers all existing stored data.
2. Create `abi-baselines/<contract-name>.breaking.txt` with a short reason.
3. Run CI — the script will update the baseline and exit 0.
4. Commit the updated JSON and the `.breaking.txt` file together.
5. Remove `.breaking.txt` in the next PR so accidental future breaks are caught.

## Running locally

```bash
# After building WASM (./scripts/build.sh):
python3 scripts/check_wasm_exports.py

# Point at a custom WASM directory:
python3 scripts/check_wasm_exports.py --wasm-dir path/to/wasm/files
```
